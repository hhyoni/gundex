# GunDex — 총기 백과사전 앱 설계 문서

작성일: 2026-06-20

## 1. 개요

GunDex는 실제 총기 모델을 **분류·검색·비교**해서 보여주는 **교육용 레퍼런스(도감) 앱**이다.
포켓몬 도감처럼, 총기 모델을 찾으면 사진과 제원·설명을 보여준다. 정보의 성격은 위키피디아의
총기 문서나 도서관의 무기 도감과 동일하다.

### 목적 / 제약 / 성공 기준

- **목적:** 코딩 공부 + 친구들과 공유 (앱스토어 출시는 하지 않음)
- **공유 방식:** 웹 링크 또는 APK 파일 (Expo가 둘 다 지원)
- **데이터 전략:** 1단계는 직접 만든 JSON, 2단계에서 위키 API 연동
  (그래서 "데이터를 가져오는 부분"을 화면과 분리해 갈아끼울 수 있게 설계)
- **성공 기준:** 목록·검색·상세·비교 4개 기능이 끝까지 동작하는 앱을 완성하고
  친구에게 링크/파일로 공유할 수 있다.

## 2. 기술 스택

**선택: Expo (React Native), 언어는 JavaScript로 시작.**

| 후보 | 장점 | 단점 | 결론 |
|---|---|---|---|
| **Expo (React Native)** | 코드 한 벌로 Android/iOS/웹, Expo Go로 폰에서 즉시 테스트, JS라 한국어 자료 많음, 핫 리로드 | JS 생태계가 처음엔 복잡, 앱 용량 큼 | ✅ 채택 |
| Flutter | UI 예쁨, 포트폴리오용 | Dart 언어, 입문 자료 적음, 진입장벽 높음 | 미채택 |
| 순수 웹앱 | 가장 단순, 링크 하나로 공유 | "진짜 앱" 느낌 덜함 | 미채택 (Expo가 웹도 커버) |

> 익숙해지면 TypeScript로 전환 가능. 처음엔 JavaScript로 시작한다.

## 3. 데이터 출처

- **사진:** Wikimedia Commons (대부분 무료·합법 라이선스, 출처 표시 의무)
- **설명:** 위키피디아
- **제원:** 위키데이터 / 위키피디아
- 라이선스 의무 때문에 스키마에 `imageCredit`(사진 저작자/라이선스)와 `sourceUrl`(출처)을 둔다.

## 4. 데이터 스키마

총기 한 개 = JSON 객체. 전체 데이터는 이 객체들의 배열(`guns.json`).

```json
{
  "id": "ak-47",
  "name": "AK-47",
  "manufacturer": "Kalashnikov Concern",
  "country": "소련 (러시아)",
  "category": "rifle",
  "caliber": "7.62×39mm",
  "weight": "3.47 kg",
  "length": "880 mm",
  "year": 1949,
  "description": "미하일 칼라시니코프가 개발한 돌격소총. 단순한 구조와 높은 내구성으로 세계에서 가장 많이 생산된 소총 중 하나다.",
  "imageUrl": "https://upload.wikimedia.org/.../AK-47_type_II.jpg",
  "imageCredit": "Photo by ___ / Wikimedia Commons, CC BY-SA 3.0",
  "sourceUrl": "https://ko.wikipedia.org/wiki/AK-47"
}
```

### 필드 설명

| 필드 | 타입 | 필수 | 이유 |
|---|---|---|---|
| `id` | 문자열 | ✅ | 각 총의 고유 코드 이름표. 검색→상세 이동, 비교에서 총을 구분할 때 필요. 사람이 보는 이름과 분리. |
| `name` | 문자열 | ✅ | 화면에 보여줄 모델명. |
| `manufacturer` | 문자열 | ✅ | 제조사. |
| `country` | 문자열 | ✅ | 제조국. |
| `category` | 문자열(정해진 값) | ✅ | 분류. 정해진 값만 사용해 분류별 목록이 깔끔히 동작. |
| `caliber` | 문자열 | ✅ | 구경(예: 9×19mm). 기호·단위 때문에 문자열. |
| `weight` | 문자열 | ✅ | 무게(예: 3.47 kg). 비교 핵심 제원. |
| `length` | 문자열 | ✅ | 전체 길이(예: 880 mm). 크기 비교용. |
| `year` | 숫자 | ✅ | 생산/등장 연도. 숫자라 정렬 가능. |
| `description` | 문자열 | ✅ | 설명(상세 본문). |
| `imageUrl` | 문자열 | ✅ | 사진 주소(썸네일+상세). |
| `imageCredit` | 문자열 | ✅ | 사진 저작자/라이선스 표시(라이선스 의무). |
| `sourceUrl` | 문자열 | ✅ | 출처 링크("더 알아보기" + 신뢰성 근거). |

### 분류(category) 정해진 값

| 코드값 | 화면 표시 |
|---|---|
| `pistol` | 권총 |
| `rifle` | 소총 |
| `shotgun` | 산탄총 |
| `smg` | 기관단총 |
| `sniper` | 저격소총 |
| `machinegun` | 기관총 |

> 코드값(`rifle`)과 표시값(`소총`)을 분리 → 나중에 영어 버전/분류 추가 시 데이터를 안 건드리고 표시만 변경.

## 5. 아키텍처

데이터와 화면을 분리한다.

```
guns.json  (데이터 파일)
    │
    ▼
데이터 함수 (gunData.js)   ← getAllGuns / getGunsByCategory / searchGuns / getGunById
    │                         (지금은 JSON, 나중엔 API — 화면 코드는 안 바뀜)
    ▼
화면들 (목록 / 검색 / 상세 / 비교)
```

- 화면은 데이터 출처를 모른다. 데이터 함수만 호출한다.
- JSON → API 교체 시 데이터 함수 내부만 바꾸면 되고 화면은 그대로.

### 네비게이션

아래쪽 탭 3개 + 각 탭 안에서 항목을 누르면 상세로(스택):

```
[ 📚 도감 ]   [ 🔍 검색 ]   [ ⚖️ 비교 ]
```

## 6. 화면 설계

### ① 분류별 목록 (도감 탭)
- 상단에 분류 필터 버튼([전체][권총][소총]…)
- `getGunsByCategory(category)`로 해당 분류만 목록 표시
- 카드 = 썸네일 + 이름 + 분류·구경, 누르면 상세로(`id` 전달)

### ② 검색 탭
- 입력창에 글자 입력 시마다 `searchGuns(text)` → 이름/제조사 부분 일치 결과 표시
- 결과 카드는 목록 카드 컴포넌트를 재사용, 누르면 상세로

### ③ 상세 화면
- `getGunById(id)`로 한 개 정보 표시 (사진 + 전체 제원 + 설명)
- `imageCredit` 표시(저작권)
- "더 알아보기" → `sourceUrl` 열기
- "비교함에 담기" → 비교 기능 입구

### ④ 비교 기능 (⚖️ 비교 탭)
- 방식: **비교함(장바구니식)**. 목록/상세에서 "비교함에 담기"로 추가.
- 총 2개가 담기면 비교 탭에서 나란히 표시. 3개째는 거부(또는 교체).
- 화면: 왼쪽에 항목 이름(분류/구경/무게/길이/연도/제조국…), 오른쪽 두 칸에 각 총의 값.
- 제원 항목은 스키마 필드를 그대로 나열 → 구현이 단순.

> "비교함" 방식 채택 이유: 상태 1개만 관리하면 되고, 어느 화면에서든 담을 수 있어 초보가 만들기 쉬움.

## 7. 범위에서 제외 (YAGNI)

다음은 1차 범위에서 제외하고, 필요 시 나중에 추가한다:
- 즐겨찾기/북마크, 로그인, 다크모드, 영어 버전
- 정렬 옵션, 비교 시 더 큰 값 강조 표시
- 위키 API 연동(2단계로 미룸)

1차 목표는 **목록·검색·상세·비교 4개 기능을 끝까지 완성**하는 것.
